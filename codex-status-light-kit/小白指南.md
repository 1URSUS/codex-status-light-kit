# Codex 状态灯小白指南

这份指南默认你在 Windows 上使用 Codex 桌面客户端或 Codex CLI，并使用 NodeMCU ESP8266 做一个 USB 状态灯。

## 你要准备什么

硬件：

| 名称 | 数量 | 备注 |
| --- | --- | --- |
| NodeMCU ESP8266 | 1 | 常见 CP2102 或 CH340 USB 转串口版本都可以 |
| 红色 LED | 1 | 表示等待确认或错误 |
| 黄色 LED | 1 | 表示 Codex 正在思考或跑工具 |
| 绿色 LED | 1 | 表示空闲或完成 |
| 220 欧姆电阻 | 3 | 每个 LED 串一个 |
| 面包板和杜邦线 | 若干 | 方便不用焊接先验证 |
| Micro USB 数据线 | 1 | 必须能传数据，不能只是充电线 |
<r 注：可直接购买红绿交通灯模组

软件：

| 软件 | 用途 |
| --- | --- |
| Arduino IDE | 烧录 ESP8266 固件 |
| Python 3 | 运行 Codex hook 脚本 |
| Codex CLI 或 Codex 桌面客户端 | 触发状态灯事件 |

## 第 1 步：接线

LED 长脚是正极，短脚是负极。每颗 LED 的长脚先串 220 欧姆电阻，再接 NodeMCU 引脚；短脚接 GND。

| 灯 | NodeMCU 引脚 | ESP8266 GPIO |
| --- | --- | --- |
| 红灯 | D1 | GPIO5 |
| 黄灯 | D2 | GPIO4 |
| 绿灯 | D7 | GPIO13 |
| 三个 LED 短脚 | GND | GND |

文字版接线图：

```text
D1 ---- 220R ---- 红 LED 长脚    红 LED 短脚 ---- GND
D2 ---- 220R ---- 黄 LED 长脚    黄 LED 短脚 ---- GND
D7 ---- 220R ---- 绿 LED 长脚    绿 LED 短脚 ---- GND
```

## 第 2 步：配置 Arduino IDE

1. 打开 Arduino IDE。
2. 进入 Preferences，把下面这个地址加入 Additional Boards Manager URLs：

```
http://arduino.esp8266.com/stable/package_esp8266com_index.json
```

3. 打开 Boards Manager，搜索并安装 `esp8266`。
4. 打开 `firmware/traffic_light_controller/traffic_light_controller.ino`。
5. 选择开发板：`NodeMCU 1.0 (ESP-12E Module)`。如果你的开发板上写的是 ESP-12F，也通常选这个；ESP-12E/ESP-12F 在这里的 Arduino 配置基本兼容。
6. 选择端口：通常是 `COM3`、`COM4`、`COM5` 之类。
7. 先点击 Verify。如果通过，再点击 Upload。

烧录完成后，绿灯应该开始闪烁，表示空闲。

如果你用的不是 NodeMCU 开发板，而是一块裸 ESP-12F 模组，则不要照抄 NodeMCU 的 D1/D2/D7 接线，需要使用 GPIO5/GPIO4/GPIO13 原始引脚，并额外接好 3.3V 供电、EN/CH_PD、GPIO0、GPIO2、GPIO15、USB-TTL 下载电路。

如果开发板背面写着“安装 CH340 驱动 / 使用 9600bps / 连接 WiFi”，可以这样理解：

| 提示 | 
| --- | --- |
| 安装 CH340 驱动 | 需要。CH340 是 USB 转串口芯片，Windows 识别 COM 口和上传固件都靠它。 |
| 本项目默认串口通信和上传都用 `115200`。 |


如果 Arduino IDE 能看到 `COM4` 这类端口，说明驱动至少已经基本工作；如果上传经常超时或端口消失，再重新安装 CH340 驱动。

## 第 3 步：安装 Python 依赖

在这个项目目录打开 PowerShell：

```powershell
python -m pip install -r .\codex_hooks\requirements.txt
```

如果 `python` 命令不存在，先安装 Python 3，并勾选 Add Python to PATH。（注：路径应确保无误）

## 第 4 步：先不接 Codex，手动测试灯

保持 NodeMCU 插在电脑上，运行：

```powershell
python .\debug_tools\test_serial_command.py IDLE
python .\debug_tools\test_serial_command.py THINKING
python .\debug_tools\test_serial_command.py WAITING_USER
python .\debug_tools\test_serial_command.py TASK_COMPLETE
python .\debug_tools\test_serial_command.py TOOL_ERROR
```

你应该看到：

| 命令 | 现象 |
| --- | --- |
| `IDLE` | 绿灯闪烁 |
| `THINKING` | 黄灯闪烁 |
| `WAITING_USER` | 红灯常亮 |
| `TASK_COMPLETE` | 绿灯常亮，约 60 秒后变回绿灯闪烁 |
| `TOOL_ERROR` | 红灯闪烁 |

如果找不到串口，先在设备管理器里看 NodeMCU 是哪个 COM 口，然后运行：

```powershell
setx STATUS_LIGHT_PORT COM5
```

把 `COM5` 换成你自己的端口。重新打开 PowerShell 后再测试。

## 第 5 步：把状态灯接入 Codex

最简单做法是让安装脚本帮你生成用户级 `hooks.json`：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\install_windows.ps1 -InstallPythonDeps
```

这个脚本会写入：

```text
%USERPROFILE%\.codex\hooks.json
```

如果你原来已经有 `hooks.json`，脚本会先备份一份 `hooks.json.bak-时间戳`。

## 第 6 步：让 Codex 信任 hook

Codex 会要求你审查本地 hook，这是正常安全机制。

在 Codex CLI 里：

1. 启动一个新 Codex 会话。
2. 输入 `/hooks`。
3. 找到 `send_signal.py` 相关 hook。
4. 选择 trust 或 enable。

在 Codex 桌面客户端里：

1. 重启 Codex，或至少开启一个新线程。
2. 如果界面提示 hook 需要审查，按提示信任。
3. 做一个会触发工具调用的任务，例如让 Codex 读目录或运行测试。

## 状态怎么对应 Codex

| Codex 事件 | 灯效 |
| --- | --- |
| `SessionStart` | 绿灯闪烁 |
| `UserPromptSubmit` | 黄灯闪烁 |
| `PreToolUse` | 黄灯闪烁 |
| `PermissionRequest` | 红灯常亮 |
| `PostToolUse` 失败 | 红灯闪烁 |
| `Stop` | 绿灯常亮 |

说明：Codex 当前使用 `PostToolUse` 表示工具调用结束；本项目会尽量从 hook 输入里判断工具是否失败。如果某个 Codex 版本没有提供失败字段，工具失败灯可能不会触发，但其他状态仍然可用。

## 常见问题

### 灯完全不亮

检查 LED 方向。长脚接 GPIO，短脚接 GND。还要确认每颗 LED 都串了 220 欧姆电阻。

### Arduino 上传失败

换一根能传数据的 USB 线，或者在 Arduino IDE 里重新选择 COM 口。

### 上传时报 could not open port COMx 或 OSError 121

这不是代码问题，是 Windows 没有成功打开 USB 串口。按下面顺序排查：

1. 关闭 Arduino IDE 右上角的 Serial Monitor 和 Serial Plotter。
2. 关闭 VSCode、串口调试助手、正在运行的 Python 测试脚本，以及可能占用同一个 COM 口的 Codex hook。
3. 拔掉 NodeMCU，等 5 秒，再插回电脑。不要插 USB Hub，优先插主板 USB 口。
4. 打开 Windows 设备管理器，展开 Ports，确认 NodeMCU 当前到底是哪个 COM 口。
5. Arduino IDE 里重新选择 Tools > Port。
6. Arduino IDE 里把 Tools > Upload Speed 改成 `115200`。
7. 开始 Upload 后，如果一直连不上，按一下 NodeMCU 的 `RST` 键；仍不行就按住 `FLASH`，点 Upload，看到开始写入后松开。
8. 换一根明确能传数据的 USB 线。很多 Micro USB 线只能充电。
9. 如果设备管理器里反复消失或显示感叹号，安装对应 USB 转串口驱动：CH340 板装 CH340 驱动，CP2102 板装 Silicon Labs CP210x 驱动。

如果错误里写的是 `Serial port COM4`，但设备管理器里 NodeMCU 已经变成 `COM5`，就说明端口号变了，重新选择新的端口即可。

### 设备管理器里只看到蓝牙串口，看不到开发板

如果 Ports 里只有“蓝牙链接上的标准串行 (COMx)”，那些不是 NodeMCU/ESP8266 开发板。不要在 Arduino IDE 里选择这些蓝牙 COM 口。

正确识别后，设备管理器里通常会出现类似这些名字：

```text
USB-SERIAL CH340 (COMx)
USB2.0-Serial (COMx)
CH340 (COMx)
Silicon Labs CP210x USB to UART Bridge (COMx)
```

如果完全没有这些设备，按这个顺序处理：

1. 换一根明确能传数据的 Micro USB 线。只充电线会亮灯，但不会出现 COM 口。
2. 换电脑上的另一个 USB 口，优先使用主板自带 USB 口，不要用 USB Hub。
3. 插拔开发板时观察设备管理器是否刷新，或者是否出现“其他设备/未知设备/USB2.0-Serial”。
4. 安装或重装 CH340/CH341 驱动。你的开发板背面写 CH340，就应该装 CH340 驱动。
5. 如果装完驱动仍没有任何新设备，检查数据线、开发板 USB 口焊接、电脑 USB 口，或者换一块开发板验证。

只有当设备管理器里出现真正的 `USB-SERIAL CH340 (COMx)` 之类端口后，Arduino IDE 里的 Port 才应该选择那个 COM 口。

### 怎么看 CH340 驱动是否装好了

图形界面看法：

1. 打开设备管理器。
2. 展开 Ports，也就是“端口 (COM 和 LPT)”。
3. 插上开发板，观察是否新增 `USB-SERIAL CH340 (COMx)`、`USB2.0-Serial (COMx)` 或类似设备。
4. 如果新增设备在“其他设备”里，并且带黄色感叹号，说明系统看到了硬件，但驱动没正确匹配。
5. 如果插拔开发板时设备管理器完全没有任何变化，优先怀疑 USB 线不是数据线、USB 口问题、开发板 USB 口问题。

命令行看法：

```powershell
pnputil /enum-drivers | Select-String -Pattern "ch340|ch341|wch" -Context 2,4
```

如果能看到 `ch341ser.inf`、`Provider Name: wch.cn`、`Class Name: Ports`，说明 CH340/CH341 驱动包已经安装过。但驱动装过不等于开发板当前已经连接成功；当前连接成功仍然要看设备管理器里有没有真正的 `USB-SERIAL CH340 (COMx)`。

### Verify 提示 ArduinoJson.h 找不到

新版固件已经不需要 ArduinoJson。确认你打开的是这个文件：

```text
firmware\traffic_light_controller\traffic_light_controller.ino
```

并确认文件第一行不是 `#include <ArduinoJson.h>`。如果 Arduino IDE 里还显示这一行，关闭当前窗口，重新打开本项目里的 `.ino` 文件。

### Python 提示没有 serial

运行：

```powershell
python -m pip install pyserial
```

### 手动测试可以，Codex 不触发

检查这几件事：

1. 是否重启了 Codex 或开启新线程。
2. CLI 里是否用 `/hooks` 信任了 hook。
3. `codex features list` 里 hooks 是否启用。新版本里 hooks 默认启用。
4. `%USERPROFILE%\.codex\hooks.json` 里的 `send_signal.py` 路径是否正确。

### 自动选错串口

指定串口：

```powershell
setx STATUS_LIGHT_PORT COM5
```

然后重启 Codex。

### 我不想真的控制硬件，只想看 hook 是否触发

开启模拟模式：

```powershell
setx STATUS_LIGHT_SIMULATE 1
```

之后 hook 会写日志，不会发送串口。日志在：

```text
codex-status-light-kit\logs\status_light.log
```

关闭模拟模式：

```powershell
setx STATUS_LIGHT_SIMULATE 0
```

## 你可以怎么改灯效

改 `codex_hooks/hook_mapping.json` 即可，例如把 `PermissionRequest` 改成黄灯闪烁：

```json
{
  "PermissionRequest": "THINKING"
}
```

常用状态只有这些：

```text
IDLE
THINKING
WAITING_USER
TASK_COMPLETE
TASK_FAILED
TOOL_ERROR
```

## 什么时候不适合用

这个状态灯依赖本机 USB 串口，所以适合 Codex CLI、Codex 桌面客户端、本机 IDE 场景。不适合纯 Codex Cloud，因为云端运行环境访问不到你电脑上的 ESP8266。
